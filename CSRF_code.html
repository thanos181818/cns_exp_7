What is CSRF?

Cross-Site Request Forgery (CSRF) is an attack that forces an end user to execute unwanted actions on a web application in which they're currently authenticated.

Attack Vectors Demonstrated:

  • GET Request: Hidden form submission via GET
  • POST Request: Form submission via POST method
  • Image Tag: CSRF via <img> tag src attribute
  • Auto-Submit: Silent attack on page load
// Vulnerable DVWA CSRF Code
if( isset( $_GET[ 'Change' ] ) ) {
  $pass_new = $_GET[ 'password_new' ];
  $pass_conf = $_GET[ 'password_conf' ];
  // No CSRF token validation!
  change_password($pass_new);
}

Mitigation Strategies:

  • Implement CSRF tokens
  • Use SameSite cookies
  • Require re-authentication for sensitive actions
  • Check Referer headers
  • Use custom request headers
